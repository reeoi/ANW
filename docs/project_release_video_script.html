Product Release Script

ANP 本地小说生产系统

一条从题材、生成、审查到成稿管理的本地流水线，让短篇和长篇创作都能被看见、被追踪、被持续优化。

3-4 分钟适合项目发布、演示录屏和社媒介绍。
产品发布面向想了解项目能力的用户和观众。
本地优先突出数据、配置和审核流程都在本机掌控。
双流程同时覆盖短篇小说和长篇小说。

很多 AI 写作工具的问题，是过程不可控。你只看到一个结果，却看不到选题依据、阶段产物、审查意见、失败原因和成本变化。ANP 把这些都拆开，让每一步都能保存、查看和继续优化。

口播
长篇小说更适合持续管理。ANP 支持创建长篇项目，生成题材定位、世界观、角色、势力、关系、大纲、卷纲和章节细纲。写正文时，每章都会保留草稿、润色、审查、修改和长期记忆。

口播
审核不是一个黑盒。短篇和长篇都可以进入 AI 审查与人工复核：哪里节奏慢，哪里人设不稳，哪里需要补充细节，都能形成可执行建议。你可以继续改，也可以保留人工判断。
画面
展示作品详情、审查结果、修改建议、人工通过或拒绝操作。

ANP 的另一个重点是本地优先。数据库、日志、提示词和中间稿都保存在本机。API Key 通过配置或环境变量注入，界面不会展示敏感信息，也不托管第三方账号流程。
画面
切到设置页，展示模型供应商、Base URL、模型识别、预算配置和日志路径说明。
03:15-03:45
口播
